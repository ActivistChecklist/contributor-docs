If you are a coder and want to help with the site, you can [fork our GitHub repository](https://github.com/ActivistChecklist/ActivistChecklist/fork) and submit a pull request with your changes.

## Setting up your local development environment

For instructions on cloning the repo, installing dependencies, and running the app locally, see the [CONTRIBUTORS.md](https://github.com/ActivistChecklist/ActivistChecklist/blob/main/CONTRIBUTORS.md) file in the main repository.

## Contributing under an anonymous identity

To decide whether you need to keep your coding work separate from your real identity, read [Protecting your privacy](/getting-started/protecting-your-privacy/). If you do, follow [Create an Anonymous GitHub Account](/reference/anonymous-github/) and [Contributing Code Anonymously](/coding/anonymous-commits/).

## Where to start

Not sure where to start? Use both of these:

- **[AC.org task board](https://github.com/orgs/ActivistChecklist/projects/2)** — Organization-wide board for Activist Checklist work. Look in the **Ready** column for issues maintainers have marked as ready to pick up.
- **[Open issues](https://github.com/ActivistChecklist/ActivistChecklist/issues)** on the main app repository — Full list if you want to search or filter directly.

If an issue description is too brief or unclear, ask for clarification in the issue thread before you start coding.

## Recommended workflow

<Steps>
1. **Pick an issue**

   Start from the **Ready** column on the [AC.org task board](https://github.com/orgs/ActivistChecklist/projects/2), or browse [open issues](https://github.com/ActivistChecklist/ActivistChecklist/issues) on the main repository. Choose something that matches your skills and availability.

   If there is no assignee yet, leave a comment to say you are working on it so effort does not get duplicated.

1. **Fork the repo**

   Create your own copy of the main Activist Checklist repository:

   <LinkButton href="https://github.com/ActivistChecklist/ActivistChecklist/fork" icon="external">
     Fork the main repository
   </LinkButton>

1. **Create a branch and make changes**

   Create a short branch name that describes your fix or feature, then implement and test your changes.

   Keep pull requests focused on one issue when possible. Smaller PRs are easier and faster to review.

1. **Open a pull request**

   Open a PR from your branch to `ActivistChecklist/ActivistChecklist`.

   In the PR description, explain what changed, why it changed, and link the issue it resolves.

1. **Address review feedback**

   We may request edits before merging. Push follow-up commits to the same branch and we will continue review in the same PR thread.
</Steps>
