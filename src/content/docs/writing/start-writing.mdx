Use this page to pick how you'd like to contribute to the **content** of the site.

Please read the [code of conduct](/getting-started/code-of-conduct/) before you start.

## Choose how you want to contribute

### Suggest edits for clarity

Improve on existing guides and content: clearer wording, simpler explanations, or better examples.

**How to contribute:** Submit changes through the [visual editor](/writing/visual-editor/) if you prefer not to use Git. (Or, if you are already comfortable with Git and MDX, edit the content in your [fork of the repository](https://github.com/ActivistChecklist/ActivistChecklist/fork) and open a pull request instead.  

### Review content for accuracy

If you have experience in **digital security**, you can help check whether advice, facts, examples, resources, and framing match real-world practice.

**How to contribute:** You submit edits the same way as above: [visual editor](/writing/visual-editor/) or through a pull request.

### Suggest a new guide

New topics and large rewrites go through our planning process so we align on the direction before you invest in a full draft.

**How to contribute:** Read [writing a guide from scratch](/writing/writing-a-guide-from-scratch/) and the [contributor style guide](/writing/style/) before you propose or write a new guide.
